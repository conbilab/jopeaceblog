# 조피스의 기록 — 디자인 브리프 & 기술 결정 (리서치 종합)

> 5개 병렬 리서치 에이전트(디자인 트렌드 · 정보구조 · 비주얼 레퍼런스 · 기술/보안 · 브랜드/콘텐츠)의 결과를 종합한 확정안.

## 1. 한 줄 정의
**조피스(조재빈)의 공개 일기 — 일상·작업·생각·맛집·여행·성취를 기록해 "자랑"이 아니라 "꾸준함과 진심"으로 신뢰를 쌓는 퍼스널 브랜딩 블로그.** 명함에 적어 "이 블로그를 보면 저를 아실 수 있습니다"가 성립하는 것이 목표.

## 2. 무드 & 비주얼 방향
- **리드 무드: Editorial Journal(에디토리얼 저널)** — 따뜻한 종이 질감, 잡지 같은 위계, 넉넉한 여백, 큰 사진. 여기에 **Warm Modern**의 온기(테라코타·골드)를 섞는다.
- **팔레트(60/30/10 규칙, 크림 우세)**
  - 크림 배경 ~60%: `#fbfaf7` / `#f3f1ea`
  - 딥그린 구조(잉크·내비·푸터·헤어라인·제목) ~30%: `#0f4536` / `#1c6b54`
  - 골드 액센트 ~10%: `#a9772f` (링크·카테고리·밑줄·active)
  - 보조 따뜻한 액센트: 테라코타 `#c45e3a` (맛집·여행 온기), 세이지 틴트 `#9caf88` (인용·칩 배경)
  - **다크모드**: 순흑 대신 딥그린 차콜 `#11201a`/`#0d1814` + 소프트 골드 `#d4a574`. 모든 색은 CSS 변수, `prefers-color-scheme` + 토글(localStorage).
- **시그니처 디테일**: 1px 점선/헤어라인 시스템(카드·구분선·날짜 컬럼) → 손맛 있는 일기 느낌(Astro Neutral 테마 패턴 차용).
- **사진 처리**: 따뜻하게 살짝 채도↓ + 미세한 필름 그레인 오버레이(저투명 SVG 노이즈), 일관된 코너 처리. 폰 사진 품질이 들쭉날쭉해도 톤이 통일되게.
- **레이아웃**: 비대칭/벤토 그리드 — 글 유형마다 카드 크기를 달리. 홈은 차분한 2~3열, 모바일 1열.

## 3. 타이포그래피 (한글 가독성 수치 고정)
- **본문/UI: Pretendard**(고딕, self-host). 긴 한글 본문은 절대 명조로 깔지 않음.
- **디스플레이/세리프 액센트: Fraunces(라틴, variable, self-host)** — 워드마크·큰 라틴 제목·날짜·숫자·키커·풀인용 부호. (한국형 에디토리얼 정석: 한글은 Pretendard 헤비, 라틴 포인트만 세리프.)
- **한글 본문 토큰**: 17px / 행간 1.7(165%) / 자간 -0.02em / 단락 간격 1.4em / **본문 줄폭 640~680px(≈ 한글 40자, WCAG CJK 권고)**. 제목은 행간 1.2~1.3.
- 타입 스케일 1.25, `clamp()` 유체 타이포로 모바일~데스크톱 일관.

## 4. 정보구조(IA) — "큐레이션 우선" 2단 구조
- **상단 내비(7개 이하)**: 홈 · 기록 · 소개 · Now · 이력 · 방명록 · (검색/다크모드). 검색·RSS·태그는 글로벌 유틸.
- **6개 카테고리(글 1개 = 카테고리 1개)** + 따뜻한 한글 코너명:
  | key | 카테고리 | 코너명 | 한 줄 콘셉트 |
  |---|---|---|---|
  | work | 작업 | **만드는 시간** | 무엇을·어떻게·왜 만들었는지. 잘된 것도 망한 것도 다 적습니다. |
  | thought | 생각 | **문득** | 정리 안 된 생각도 일단 적어둡니다. |
  | daily | 일상 | **오늘의 결** | 대전에서 1인 기업으로 사는 평범한 하루. |
  | food | 맛집 | **맛의 기록** | 작업이 잘되는 자리, 오늘 뭘 먹었나. |
  | travel | 여행 | **바람 따라** | 강의 출장길에 주운 낯선 장면들. |
  | win | 성취 | **발자국** | 수상·언론·마일스톤. 자랑이 아니라 증거로. |
- **태그**는 카테고리 위에 자유롭게(#대전 #AI영상 #강의후기 #브랜딩 …). `/tags` 자동 인덱스.
- **완성도 라벨(디지털 가든)**: 🌱 단상 / 🌿 정리 중 / 🌳 완성 — 가벼운 글의 발행 장벽을 낮춤.
- **고정 페이지**: 소개(About, 골든서클 Why→How→What + 오리진 스토리) · Now(요즘 근황, 자주 업데이트) · 이력(세로 타임라인) · 방명록(giscus) · 연락(명함 랜딩 포함) · uses(장비·툴, 선택) · archive(연도별).

## 5. 기술 스택 (확정) — 정적 = 최소 공격면
- **Astro 7 (output: static)** + **마크다운/MDX 콘텐츠 컬렉션 + Zod 스키마**(프런트매터 오타 시 빌드 실패 → 깨진 글 방지).
- **글쓰기 GUI: Sveltia CMS @ `/admin`** (Decap 드롭인 대체). 이유: Decap은 2025-09 XSS 미패치 + 2026-01 프록시 취약점으로 유지보수 중단 상태. Sveltia는 동일 `config.yml`, **자체 GitHub OAuth 내장(Vercel용 별도 OAuth 프록시 불필요)**, 600KB, **모바일 작성 지원**(폰에서 일기 기록), <24h 버그픽스, editorial workflow(초안→리뷰→발행).
- **방명록/댓글: giscus**(GitHub Discussions). GitHub 로그인 필수 = 익명 스팸 원천 차단, DB 없음, GitHub에서 모더레이션.
- **이미지: astro:assets/Sharp** — 빌드타임 AVIF/WebP/JPEG 반응형, 런타임 비용 0.
- **호스팅: Vercel(정적)** + GitHub. 월 $0, 서버/DB 없음.
- **보안 헤더(vercel.json, 전 경로)**: HSTS(2년, preload) · X-Content-Type-Options nosniff · X-Frame-Options DENY · Referrer-Policy strict-origin-when-cross-origin · X-XSS-Protection 0 · Permissions-Policy(카메라/마이크/위치 차단) · **엄격한 CSP**(script-src에 'unsafe-inline' 금지; giscus.app·api.github.com만 허용).
- 운영 보안: GitHub 2FA, main 브랜치 보호, 최소 OAuth scope, securityheaders.com A등급 확인.

## 6. 톤앤매너 (콘텐츠 룰)
- 존댓말 에세이체(~합니다/~했어요), 1인칭 "저", 자기호칭 "조피스". 짧은 문단(모바일).
- **정직 원칙**: 모든 작업기에 "잘 안 된 점" 한 단락 의무화. 실패·비용 숨기지 않기.
- **독자를 주인공으로**: 후기/성과는 고객·수강생 before→after 중심, 조피스는 "옆에서 도운 사람".
- 자랑은 사실+과정으로 대체. 신뢰 배지(중도일보·프로젝트100·1% 기부·영상 500편)는 각주처럼 담백하게.
- 글 끝엔 항상 독자에게 건네는 한 문장.

## 7. 모션
- "less but meaningful": 스크롤 페이드/스태거, 링크 밑줄 슬라이드, 카드 호버 리프트, 절제된 스크롤 진행바. 비즈니스 랜딩의 커스텀 커서/마그네틱/오브는 **읽기 페이지에서 제거**. `prefers-reduced-motion` 필수 존중.
